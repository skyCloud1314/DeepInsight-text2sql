# 异常检测显示清晰化设计文档

## 概述

本设计文档描述如何改进异常检测模块的前端显示，使用户能够一眼看出具体的异常内容，而不仅仅是异常数量统计。

## 架构

### 当前架构问题
- 异常摘要只显示数量统计（总异常数、高风险、中风险）
- 用户需要展开详情才能了解具体异常
- 缺乏异常类型的直观标识
- 没有异常影响程度的可视化

### 改进架构
```
异常检测结果
├── 增强异常摘要区域
│   ├── 异常统计数据
│   ├── 主要异常预览
│   └── 异常类型图标
├── 异常详情卡片
│   ├── 异常原因说明
│   ├── 数据样本预览
│   └── 处理建议
└── 异常历史对比
    ├── 趋势变化
    └── 置信度显示
```

## 组件和接口

### 1. 增强异常摘要组件 (EnhancedAnomalySummary)

**功能**: 在原有统计数据基础上增加异常预览信息

**接口**:
```python
def render_enhanced_summary(anomaly_analysis: Dict) -> None:
    """
    渲染增强的异常摘要
    
    Args:
        anomaly_analysis: 异常分析结果
    """
```

**显示逻辑**:
- 保留原有的三列统计（总异常数、高风险、中风险）
- 在统计数据下方添加主要异常预览
- 使用图标和颜色区分异常类型和严重程度

### 2. 异常类型图标映射 (AnomalyTypeIcons)

**功能**: 为不同异常类型提供直观的图标标识

**映射规则**:
```python
ANOMALY_TYPE_ICONS = {
    "statistical_outlier": "📊",
    "extreme_value": "⚠️", 
    "zero_value_anomaly": "🔴",
    "negative_profit": "💸",
    "high_profit_margin": "📈",
    "price_anomaly": "💰",
    "trend_break": "📉",
    "declining_trend": "⬇️"
}
```

### 3. 异常预览卡片 (AnomalyPreviewCard)

**功能**: 在摘要区域显示异常的简要信息

**显示内容**:
- 异常类型图标
- 异常简要描述（限制字符数）
- 关键数据样本
- 影响程度指示器

## 数据模型

### 异常摘要数据结构

```python
@dataclass
class AnomalySummaryData:
    total_anomalies: int
    high_severity: int
    medium_severity: int
    low_severity: int
    primary_anomaly: Optional[AnomalyPreview]
    anomaly_types: List[str]
    confidence_score: float
    historical_comparison: Optional[HistoricalComparison]

@dataclass 
class AnomalyPreview:
    type: str
    icon: str
    short_description: str  # 限制50字符
    sample_data: List[str]  # 1-2个数据样本
    impact_level: str  # "high", "medium", "low"
    confidence: float
    quick_action: Optional[str]  # 快速处理建议
```

### UI显示优先级

1. **单个异常时**: 直接显示完整的异常预览
2. **多个同类异常时**: 显示主要异常 + "等X个同类异常"
3. **多个不同类异常时**: 显示最高风险异常 + "等X个其他异常"

## 正确性属性

*属性是一个特征或行为，应该在系统的所有有效执行中保持为真——本质上是关于系统应该做什么的正式声明。属性作为人类可读规范和机器可验证正确性保证之间的桥梁。*

### 属性 1: 异常摘要完整性
*对于任何*异常分析结果，如果总异常数大于0，则摘要区域必须显示至少一个具体的异常描述
**验证: 需求 1.1, 1.2**

### 属性 2: 异常类型图标一致性  
*对于任何*检测到的异常类型，系统必须显示对应的标准图标，且相同类型的异常必须使用相同图标
**验证: 需求 2.1, 2.2, 2.3, 2.4**

### 属性 3: 风险等级颜色映射
*对于任何*异常，其显示颜色必须与风险等级一致：高风险=红色，中风险=橙色，低风险=黄色
**验证: 需求 3.3, 3.4, 3.5**

### 属性 4: 异常原因可追溯性
*对于任何*显示的异常，用户必须能够通过UI界面了解该异常的检测原因和触发条件
**验证: 需求 4.1, 4.2, 4.3, 4.4**

### 属性 5: 数据样本真实性
*对于任何*显示的异常数据样本，必须来自实际的异常检测结果，不能是虚构或示例数据
**验证: 需求 6.1, 6.2, 6.4**

## 错误处理

### 异常数据缺失处理
- 当异常分析结果为空时，显示"未检测到异常"
- 当异常数据不完整时，显示可用信息并标注"部分信息缺失"

### 显示异常处理  
- 当异常描述过长时，智能截断并保留关键信息
- 当图标加载失败时，使用文字标签替代
- 当历史数据不可用时，隐藏对比功能
